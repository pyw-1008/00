# AGENTS.md

本文件为 Codex Code（Codex.ai/code）在本项目中工作时提供指引。

## 这是什么

这是一个现场互动墙项目，用于直播/课堂场景：观众扫码用手机投票（单选）并发送一句话留言；投影出去的大屏页实时显示票数柱状图和从右往左滚动的留言；带密码的主持人后台页负责管理当前题目。

完整的产品说明、范围、验收标准以及编号的竖切切片，以 [../docs/PRD.md](../docs/PRD.md) 为准。做任何功能前，必须先读 PRD，并对照对应切片的验收标准。

仓库目前处于**骨架阶段**：三个空页面、一个能启动的本地后端、一个已初始化的 SQLite 数据库。PRD 里的功能切片都还没实现。切片要一个一个做，每片都能独立验证；未经用户明确要求，不要跳着做，也不要提前实现后续切片。

## 常用命令

当前骨架使用 Python 3 标准库实现，不引入额外依赖。

```powershell

```

启动后访问：

- 健康检查：`http://127.0.0.1:8000/api/health`
- 数据库连通性检查：`http://127.0.0.1:8000/api/db-check`

开发环境页面：

- 观众页（手机）：`http://127.0.0.1:8000/audience`
- 大屏页（投影）：`http://127.0.0.1:8000/screen`
- 主持人后台：`http://127.0.0.1:8000/admin`

目前还没有正式测试套件或 lint。骨架阶段的手动验证标准是：后端能启动，三个页面能打开，`/api/db-check` 能向 SQLite 写入并读回一条测试数据。

## 当前架构

- **后端**：`server.py` 是入口，使用 Python 标准库中的 `http.server` 提供本地 HTTP 服务。
- **页面**：`public/` 下放置三个静态占位页：
  - `public/audience.html`
  - `public/screen.html`
  - `public/admin.html`
- **样式**：`public/styles.css`。
- **数据库**：`schema.sql` 定义 SQLite 表结构，运行时数据库文件为 `app.sqlite3`。
- **环境变量**：启动时读取环境变量，也会读取本地 `.env`。后台密码使用 `ADMIN_PASSWORD`，不得写死到代码中。

当前路由：

- `/`：重定向到 `/audience`
- `/audience`：观众页
- `/screen`：大屏页
- `/admin`：主持人后台页
- `/api/health`：后端健康检查
- `/api/db-check`：写入并读回一条测试留言，验证 SQLite 可用

当前数据库表结构见 `schema.sql`：

- `questions`：题目
- `question_options`：题目选项，外键关联 `questions`
- `votes`：投票，外键关联 `questions` 和 `question_options`
- `messages`：留言，当前也用于数据库连通性测试

## 约定与关键决策

- **以 PRD 为准**：本项目以 [../docs/PRD.md](../docs/PRD.md) 为准，每个功能切片都必须对照 PRD 的验收标准来实现和验证。
- **实时策略使用轮询**：大屏页/观众页后续做实时刷新时，每约 1.5 秒向后端轮询一次。除非 PRD 明确改变，不要引入 WebSocket 或 SSE。
- **数据存 SQLite**：投票和留言先存住、能读回来即可；刷新页面后数据仍在。骨架和早期切片阶段，不额外追求超出 PRD 的持久化严谨性。
- **每加一个功能，配一个对应测试**：新增功能必须有能验证该切片验收标准的测试或明确的验证方式。
- **同一时刻只有一道生效题目**：后续题目管理由主持人后台控制，具体实现以 PRD 切片为准。
- **后台密码来自环境变量**：后台密码从 `.env` 或系统环境变量中的 `ADMIN_PASSWORD` 读取。绝不写死进代码，绝不提交真实 `.env`。
- **范围刻意做小**：按 PRD「明确不做」执行，例如 ≤100 人同时在线、单服务器、观众无需登录、不做严格的一人一票。
- **界面文案和代码注释用中文**：新增用户可见文案、必要注释和项目说明尽量使用简体中文，并与现有文件风格保持一致。

## Git 与本地文件

- `*.pyc`
